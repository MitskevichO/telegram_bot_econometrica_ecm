Разработанный телеграм-бот предназначен для автоматизации полного цикла эконометрического анализа временных рядов. Его архитектура строится на принципах модульности и интерактивности: пользователь взаимодействует с ботом через кнопочное меню, а все расчёты выполняются на серверной стороне с использованием специализированных библиотек Python. Для реализации проекта выбраны следующие библиотеки, являющиеся бесплатными и широко распространенными для расчётов:
	Аiogram – это библиотека для создания телеграм-ботов. Она берёт на себя всю работу по приёму сообщений, обработке нажатий на кнопки и запоминанию, на каком шаге диалога находится пользователь. Эта библиотека достаточно производительная, хорошо задокументирована и поддерживает все актуальные функции Telegram.
	Statsmodels – библиотека используемая непосредственно для эконометрического моделирования. Содержит реализацию теста ADF, линейной регрессии, функций для работы с временными рядами. Именно она используется для оценки (pvalues) коинтеграционного уравнения, построения ECM и расчёта тестов.
	Рandas – используется для работы с табличными данными. Позволяет загружать и читать Excel-файлы, автоматически определять колонку с датами, преобразовывать числовые ряды (например, вычислять первую разность для ECM) и создавать лаги переменных. Она также выполняет интерполяцию пропущенных значений, что позволяет избежать потери наблюдений. Подготовленные таким образом данные затем передаются в statsmodels для оценки моделей.
4.	Matplotlib и Seaborn – библиотеки для визуализации. Они генерируют графики временных рядов, коррелограммы (ACF/PACF), графики стационарности рядов, остатков модели. Изображения отправляются пользователю в виде фотографий в чат и встраиваются в итоговый Excel-отчёт.
5.	Openpyxl – позволяет создавать Excel-файлы, добавлять в них изображения и формировать многостраничные отчёты.
6.	Aiocron – библиотека позволяющая отслеживать пользователей, которые долго бездействовали. (В данном проекте проверяет каждые 5 минут сессии пользователей и удаляет те, которые не использовались более 30 минут.)
Немаловажную роль играет и структура проекта, которая позволяет быстро ориентироваться и расширять функциональность кода. Структура данного проекта имеет следующий вид:
	Свободные файлы: Main.py – точка входа: создаёт экземпляр бота, диспетчера, настраивает хранилище сессий (InMemorySessionStore) и запускает поллинг (запрос у Telegram новых сообщений). Config.py – загрузка настроек (токен бота, параметры сессий, пути к папкам).
	Папка routers (маршрутизатор диалогов): сommon.py – является единственным файлом-роутером, который определяет сообщение или какая кнопка пришли от пользователя, и направляет их к нужной функции. В данном файле сосредоточены все обработчики команд и callback-запросов. 
	Папка keyboards (формирование клавиатур): Каждый файл возвращает inline-клавиатуры (кнопки под сообщением) или reply-клавиатуры (обычные кнопки внизу экрана).  
	 main_menu.py – создает главное меню с загрузкой файла и инструкцией его загрузки.
	roles.py – меню после загрузки данных: клавиатура для выбора зависимой и независимой переменных из числовых колонок загруженного файла.
	analysis_menu.py – отображает возможности построения эконометрической модели такие как, проверка на коинтеграцию, построение ECM, настройки лагов, диагностика, прогноз.
	tests_menu.py – меню диагностических тестов (Уайт, Бройша–Годфри, Чоу, запуск всех тестов одновременно).
	Папка states содержит файл states.py, где описаны состояния диалога (ожидание файла, выбор переменных, настройки и т.д.). Благодаря этому бот запоминает, на каком шаге находится пользователь, и реагирует только на те команды, которые соответствуют текущему шагу.
	Папка srs является ядром эконометрических расчётов. Содержит модули:
	validators.py – загрузка и проверка Excel файла. определяет колонку с датами (по ключевым словам или преобразованием в datetime), интерполирует пропуски, проверяет, что есть хотя бы две числовые колонки. Возвращает структуру ValidationResult. 
	stationarity.py – ADF-тест и определение порядка интеграции.
	cointegration.py –  реализует двухшаговую процедуру Энгла–Грейнджера: линейная регрессия для долгосрочного уравнения, затем проведение ADF-тестирование по остаткам. 
	models.py – строит ECM: берёт остатки из коинтеграции, добавляет лаги ΔY по выбору, оценивает также линейную регрессию. Возвращает объект ECMResult (объект, в котором собраны все ключевые параметры последней построенной ECM-модели) с коэффициентами и p‑values. 
	diagnostics.py – написаны тесты Уайта, Бройша–Годфри, Чоу. Для первых двух строятся согласно теоретической процедуре проведения теста вспомогательные регрессии, для Чоу идет разбиение выборки и расчёт F статистики.
	forecast.py – рассчитывается точечное прогнозирование и его доверительный интервал: для линейной регрессии используется аналитический интервал, для ECM – интервал на основе стандартной ошибки регрессии σ ̂_u с поправкой на поправочный коэффициент √h.
	visualizer.py – генерация всех графиков (временные ряды, ACF/PACF, стационарность, остатки) сохраняются в формате PNG через matplotlib.
	reports.py – формирование Excel-отчёта с листами результатов и встроенными изображениями.
	interpreter.py – текстовое оформление результатов для вывода в чат пользователю.
	Папка utils (вспомогательные утилиты):
	cache.py – этот модуль отвечает за временное хранение данных пользователя между разными действиями в боте. Поскольку бот не использует базу данных (всё хранится в оперативной памяти), здесь реализован словарь, который автоматически отчищает старые записи. При этом максимальное время активности сессии без активности составляет 30 минут TTL (Time To Live).
	file_handler.py – осуществляет приём и временное сохранение загруженных файлов от пользователя (в байтовом виде).
	logger.py – настраивает формат логов в консоли: выводит время, уровень сообщения (INFO, ERROR), имя модуля и сам текст. Это помогает разработчику отслеживать работу бота и находить ошибки.
	scheduler.py – использует aiocron для фонового запуска задачи cleanup_expired каждые 5 минут. Это позволяет автоматически отслеживать и удалять сессии, которые не использовались дольше TTL, и тем самым экономить память.
Такая структура обеспечивает разделение ответственности и позволяет разработчику быстро ориентироваться в коде.  Это упрощает доработку и добавление новых возможностей. По итогу диалоговая логика находится в папке routers, расчёты в папке src, интерфейсные элементы в keyboards, а вспомогательные задачи в utils.
